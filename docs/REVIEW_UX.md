# UX 리뷰 — Android Markdown Viewer + Mermaid
- 리뷰어: UX Agent (모바일 UX 전문가 관점)
- 일자: 2026-01-27

---

## 1. UX 플로우 평가

### 강점
- 4화면 구조(Home/Viewer/FocusView/Settings)가 명확하고 단순
- Home → Viewer → Focus View 드릴다운 패턴이 자연스러움
- 최근 문서 즉시 접근 가능

### 개선 필요
- **TOC UI가 구체화되지 않음**: 바텀시트/사이드 드로어/새 화면 중 미결정
- **상단바 혼잡**: 제목 + 검색 + TOC + 설정 = 4개 요소가 좁은 모바일 화면에 과다
- **파일 선택 UX 미명시**: SAF 직접 사용 vs 인앱 파일 브라우저 여부

## 2. Focus View 개선 권고

### 현재: 탭 → 확대, pinch zoom + pan, 닫기 버튼
### 추가 필요:
- **Pull-to-dismiss** (전체화면 뷰의 표준 패턴)
- **Android predictive back gesture** 지원
- **다중 다이어그램 간 스와이프 내비게이션** (좌우 스와이프로 이전/다음)
- **초기 줌 레벨 전략 명시** (fit-to-width vs fit-to-screen)

## 3. 자동 방향 전환의 UX 영향

### 긍정적
- 모바일 세로 모드에서 LR 다이어그램 가독성 문제를 적극 해결

### 우려
- **예측 불가능성**: 사용자가 "변조된" 다이어그램을 보게 되어 혼란 유발
- **복잡도 조건(edge>=6 등)이 투명하지 않음**

### 필수 개선
1. 변환된 다이어그램에 **시각적 인디케이터** 표시 (예: "자동 조정됨" 뱃지)
2. **원본 보기 옵션** (롱프레스 또는 컨텍스트 메뉴)
3. 설정을 "항상 / 복잡한 것만 / 절대 안 함" **3단계**로 확장
4. **첫 실행 시 온보딩** 안내

## 4. 접근성 평가

### 충족
- 폰트 스케일 0.9~1.4
- 다크모드 대비

### 부족
- **TalkBack 지원 계획 없음** — 다이어그램 alt text/설명 제공 방법 미정
- **색각 이상(Color blindness)** 대응 미고려
- **터치 타겟 48dp 이상** 보장 여부 미명시
- **시스템 폰트 크기 설정** 존중 여부 불명확 (앱 내 설정만으로 제한?)

## 5. 누락된 UX 고려사항

### 콘텐츠 관리
- 읽기 위치 저장/복원 (문서 재오픈 시)
- 북마크/즐겨찾기
- 최근 문서 목록 관리 (삭제, 정렬, 검색)

### WebView 기반 문제
- 로딩 상태 인디케이터
- 텍스트 선택 및 복사 가능 여부
- 매우 긴 문서에서 스크롤 성능

### 기기 대응
- 가로 모드(landscape) 및 태블릿/폴더블 지원
- 멀티윈도우/Split screen 레이아웃 동작
- 외부 키보드 연결 시 내비게이션

### 공유 기능
- 다이어그램 이미지 저장 (PNG)
- 문서 공유 인텐트
